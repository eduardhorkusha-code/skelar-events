pnpm -v
✅
Виглядає так коли все OK:
9.x.x або вище — готово, йдемо далі.

npm install -g @anthropic-ai/claude-code

claude --version
✅
Виглядає так коли все OK:
Виведе версію, наприклад 1.x.x.

крок 2.2
Клонуй репозиторій та встанови залежності
Виконай всі три команди по черзі. Після кожної чекай поки вона завершиться.
У вікні Terminal — по одній команді за раз
1. Клонуй репо (виконай один раз):
git clone https://github.com/eduardhorkusha-code/skelar-vault.git
2. Перейди в папку застосунку:
cd skelar-vault/apps/hub
3. Встанови залежності (займе ~1–2 хвилини):
pnpm install
✅
Виглядає так коли все OK:
pnpm install закінчиться словами Done in ... або Progress: resolved .... Жодних червоних ERROR.
крок 2.3
Додай секретний файл .env.local
Цей файл містить ключі до бази даних. Eduard надішле тобі окремо (не через git).
У Finder — перейди в папку skelar-vault/apps/hub/ і поклади туди файл .env.local
💡 Файли що починаються з . приховані в macOS. Щоб побачити їх у Finder: натисни ⌘ + Shift + .
Або через термінал — перевір що файл на місці:
ls -la | grep .env
✅
Виглядає так коли все OK:
Термінал покаже рядок з .env.local в переліку файлів.
🚫 Ніколи не комітити .env.local у git. Він вже в .gitignore — просто не додавай його вручну.
крок 2.4
Запусти dev-сервер і відкрий у браузері
Перевіримо що проєкт запускається.
У Terminal — переконайся що ти в папці apps/hub 
# якщо ти не в apps/hub — перейди:
cd skelar-vault/apps/hub

# запусти сервер:
pnpm dev
Відкрий браузер → адреса http://localhost:3000
✅
Виглядає так коли все OK:
У терміналі з'явиться ✓ Ready in .... У браузері відкриється логін-сторінка SKELAR Vault — залогінься через свій @skelar.tech акаунт.
💡 Сервер залишається запущеним у терміналі — не закривай це вікно поки працюєш. Щоб зупинити — Ctrl + C.

Git workflow — зберігай зміни

Після кожного блоку роботи зберігай в git і пушай на GitHub

крок 4.1
Створи свою робочу гілку (один раз)
Git-гілка — це твій власний простір для змін. Ти не впливаєш на основну гілку поки не зробиш merge.
У Terminal — з папки skelar-vault (не з apps/hub) 
git checkout -b feature/performance-review-ui
✅
Виглядає так коли все OK:
Switched to a new branch 'feature/performance-review-ui'
крок 4.2
Комітай і пушай свої зміни
Виконуй ці команди кожного разу коли хочеш зберегти прогрес. Додавай тільки папки performance-review — нічого зайвого.
У Terminal — з кореня skelar-vault/ 
# додай тільки свої файли:
git add apps/hub/performance-review/
git add apps/hub/app/performance-review/

# збережи з коментарем (заміни текст в лапках):
git commit -m "feat: employee view layout"

# відправ на GitHub:
git push origin feature/performance-review-ui
✅
Виглядає так коли все OK:
Після git push побачиш щось на кшталт Branch 'feature/...' set up to track remote branch. Зміни з'являться на GitHub.
🚫 Не додавати в git: talent-acquisition/, app/admin/, lib/supabase/, apps/reputation/ — ці папки чіпати не можна.

Access model — хто що бачить

Довідково: роль визначає що показувати у UI

РольБачитьМоже редагувати
employee Тільки свій review у поточному циклі Self-assessment поля
manager Свій + всі прямі підлеглі (via manager_id) Manager assessment + категорії
hr Всі review у поточному циклі Читання + калібрація
admin Все + управління доступом Повний доступ
ℹ️ Немає рядку в perf_review_access → редірект на /?no_access=performance-review. Доступ видає Eduard.
